## 2. 전체 기능 정리

### 2-1. 멤버 화면

#### 로그인 (`/login`)
- "3월, 44명의 멤버와 함께하고 있습니다" 배지
- 매주 수요일 저녁 8시 / 일요일 오후 3시 안내
- 멤버십 가입 버튼 (네이버 스마트스토어 연결)
- 하단 카카오톡(well__moment) + 인스타그램 문의
- 라벨 "본인 확인 코드"로 변경

#### 캘린더 (`/calendar`)
- 날짜별 예약 인원수 표시
- 휴무 날짜 "휴무" 표시 (관리자가 닫은 날짜)
- 월별 멤버십 제한 — 비활성월이면 결제 안내 모달 + 스마트스토어 링크

#### 스팟 선택 (예약)
- 스팟별 예약 인원 + 스몰토크/사색 비율 표시
- 닫힌 스팟 "휴무", 스팟별 안내 메시지 표시
- 소인원 넛지 — "소규모로 더 깊은 대화" 추천
- 마감 임박(80%) / 소규모(3명 이하) 뱃지
- 불편 멤버 회피 경고
- 사색 모드 선택 시 활동 선택 (일기 쓰기, 책 읽기, 편지 쓰기, 드로잉, 생각 정리, 그냥 쉬기)

#### 세션 준비 (`/session-ready`)
- 날짜 + 스팟 + 스몰토크/사색 인원수
- 오늘의 참가자 목록 (익명)
- 스몰토크 모드 → 오늘의 대화 카드 (랜덤 질문)
- 사색 모드 → 오늘의 사색 카드 (10가지: 읽기 명상, 자유 글쓰기, 소리 관찰 등)
- 스마트폰 보관 안내 + 정시 시작 안내
- "스마트폰을 내려놓을 준비가 되었습니다" 버튼 → 마무리 화면
- **세션 종료 후** → 인스타그램 스토리 공유 화면 (2026-04-10 추가)

#### 내 기록 (`/my-history`)
- 참석 세션 수, 방문 스팟, 쉼의 시간
- 디톡스 레벨 시스템 (시작하는 중 → 새싹 → 탐험가 → 러너 → 마스터 → 레전드)
- 인스타 스토리 공유 카드 생성 (Canvas 1080x1920)
- 불편 신고 버튼 + 모달

#### 피드백 (`/feedback`) — 탭 3개
1. **피드백** — 세션별 만족도 + 서비스 개선 + 불편사항(관리자만 확인) + 오늘의 한마디(익명)
2. **사색 회고** — 사색 참석 세션 → 느낀 점, 깨달음, 다음에 하고 싶은 것 → 나만의 사색 일지
3. **요청** — 운영자에게 바라는 점 (카테고리별) + 운영자 답변 확인
- **인스타 공유 배너** — 상단에 디톡스 이미지 공유 버튼 (2026-04-10 추가)

#### 스몰토크 가이드 (`/guide`)
- 스몰토크 팁 + 대화 카드

### 2-2. 스팟 운영자 화면 (`/admin/spot`)

## 3. "대화 열기" 기능 기획

### 배경
- 멤버들이 인기 스팟에 쏠리고, 비인기 스팟은 예약이 비는 문제
- 강제 배정 없이, 빈 스팟에 "대화 주제를 정할 수 있는 특권"을 부여하여 자발적 분산 유도

### 발동 조건
1. 같은 날짜에 다른 스팟 중 하나라도 예약 4명 이상
2. 해당 스팟의 예약이 0명
- **하이브리드 조건** (2026-04-03 개선): 4명 이상 스팟 OR 전체 예약자 6명 이상일 때 활성화

### 주요 규칙
| 항목 | 규칙 |
|------|------|
| 주제 수 | 한 세션(날짜+스팟)당 1개 |
| 주제 작성 권한 | 해당 스팟 첫 번째 예약자만 |
| 주제 입력 | 카테고리 선택 또는 직접 작성 (최대 30자) |
| 주제 입력 강제 여부 | 선택사항 |
| 주제 작성자 이탈 시 | 주제 유지, 다른 예약자 있으면 확인 메시지 |
| 조건 변동 시 | 이미 열린 주제 유지, 새 대화 열기는 비활성화 |

### 카테고리 (7개, 총 50개 추천 주제)
- 일상 수다 (7개), 취미와 취향 (7개), 여행과 장소 (7개)
- 감정과 마음 (7개), 관계와 사람 (7개), 상상과 만약에 (7개), 성장과 변화 (8개)

### DB
```sql
CREATE TABLE conversation_topics (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  date TEXT NOT NULL,
  spot TEXT NOT NULL,
  topic TEXT NOT NULL,
  category TEXT,
  created_by TEXT NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(date, spot)
);
```

### API
- `POST /api/conversation-topics` — 대화 주제 생성
- `GET /api/conversation-topics?date=` — 날짜별 주제 조회
- `GET /api/conversation-topics/suggestions` — 추천 주제 목록
