# 기획서: "대화 열기" 기능

## 1. 배경 및 목적

### 해결하려는 문제
- 멤버십 가입자들의 예약이 특정 인기 스팟에 쏠리고, 다른 스팟은 예약이 비어 있음
- 멤버들은 "사람이 많은 곳"을 기준으로 스팟을 선택하기 때문에 쏠림이 가속됨
- 비인기 스팟 운영자들의 불만 발생

### 해결 방향
- 스팟 정원 제한이나 예약 강제 없이, **비인기 스팟에 자발적으로 예약하고 싶은 이유**를 만든다
- 빈 스팟에 첫 번째로 예약하는 멤버에게 **"대화 주제를 정할 수 있는 특권"**을 부여
- 다른 멤버들이 그 대화 주제를 보고 흥미를 느껴 해당 스팟에 예약하도록 유도

### 핵심 원칙
- 멤버의 자율성을 침해하지 않는다 (스팟 강제 배정 없음)
- 타임오프클럽의 익명 원칙을 유지한다 (이름/직업/배경 노출 없음)
- 진행자 없는 자유로운 스몰토크 구조를 깨지 않는다

## 2. 기능 상세

### 2-1. 발동 조건

"대화 열기" 기능은 다음 조건이 **모두** 충족될 때 활성화된다:

1. 같은 날짜에 **다른 스팟 중 하나라도 예약이 4명 이상**
2. 해당 스팟의 예약이 **0명**

즉, 모든 스팟이 비어있는 예약 초반에는 이 기능이 나타나지 않는다.
이미 쏠림이 시작된 상황에서만 빈 스팟에 활성화되는 장치이다.

```
예시 상황:
- 약수 스티키플로어: 5명 → 대화 열기 비활성화
- 망원 다시점: 4명 → 대화 열기 비활성화
- 서촌 터틀도브: 0명 → ✅ 대화 열기 활성화
- 압구정 벤슨: 0명 → ✅ 대화 열기 활성화
```

### 2-2. 예약 화면 UI

#### 조건 미충족 시 (기존과 동일)
```
약수 스티키플로어 — 2명 예약 중
망원 다시점 — 1명 예약 중
서촌 터틀도브 — 0명
압구정 벤슨 — 0명
```

#### 조건 충족 시 (빈 스팟에 대화 열기 노출)
```
약수 스티키플로어 — 5명 예약 중
망원 다시점 — 4명 예약 중
서촌 터틀도브 — 0명
  ✨ 첫 번째로 예약하고, 나누고 싶은 대화를 열어보세요
압구정 벤슨 — 0명
  ✨ 첫 번째로 예약하고, 나누고 싶은 대화를 열어보세요
```

#### 누군가 대화를 연 후
```
약수 스티키플로어 — 5명 예약 중
망원 다시점 — 4명 예약 중
서촌 터틀도브 — 1명 예약 중
  💬 "요즘 빠져있는 넷플릭스 추천"
압구정 벤슨 — 1명 예약 중
  💬 "혼자 여행에서 생긴 에피소드"
```

### 2-3. 대화 주제 입력 UX

빈 스팟의 "대화 열기"를 선택하면 주제 입력 화면이 나타난다.

#### 화면 구성
```
어떤 이야기를 열어볼까요?
같은 이야기를 나누고 싶은 멤버들이 이 주제를 보고 모일 수 있어요

[카테고리 선택] 또는 [직접 작성하기]
```

#### 카테고리별 추천 주제 (선택형)

멤버가 카테고리를 먼저 고르고, 그 안에서 주제를 선택하는 구조.

**일상 수다** (가장 가벼운 톤)
1. 요즘 빠져있는 넷플릭스/유튜브 추천
2. 인생 최애 카페 or 단골집 자랑
3. 최근에 먹은 것 중 제일 맛있었던 거
4. 요즘 내 플레이리스트 1위 곡
5. 최근에 지른 소비 자랑 or 후회
6. 아침형 vs 저녁형, 서로의 하루 비교
7. 요즘 SNS에서 계속 보게 되는 것들

**취미와 취향** (나를 드러내는 이야기)
8. 남들한테 말하면 의외라는 나의 취미
9. 시간 가는 줄 모르고 빠져드는 것
10. 돈 안 들이고 행복해지는 나만의 방법
11. 요즘 새로 시작한 것 or 시작하고 싶은 것
12. 혼자만 알고 있던 숨은 맛집/장소 공유
13. 어릴 때는 싫었는데 지금은 좋아진 것
14. 아무 이유 없이 모으고 있는 것

**여행과 장소** (경험 나누기)
15. 혼자 여행에서 생긴 에피소드
16. 다시 가고 싶은 장소와 그 이유
17. 여행지에서 만난 인상 깊은 사람
18. 가보고 싶은 곳 버킷리스트
19. 길 잃어서 오히려 좋았던 경험
20. 살아보고 싶은 도시와 그 이유
21. 최근에 우연히 발견한 좋은 장소

**감정과 마음** (솔직한 이야기)
22. 요즘 나를 가장 많이 웃게 하는 것
23. 최근에 마음이 움직인 순간
24. 아무한테도 말 안 했던 소소한 고민
25. 나만의 스트레스 해소법
26. 요즘 자기 전에 드는 생각
27. 최근에 나 자신이 좀 대견했던 순간
28. 괜히 눈물 나게 만든 영화/음악/글

**관계와 사람** (사이의 이야기)
29. 오래된 친구와 최근에 만난 친구의 차이
30. 누군가의 한마디에 오래 남은 말
31. 어른이 되면서 달라진 우정의 기준
32. 혼자가 편한 순간 vs 누군가가 필요한 순간
33. 최근에 고마웠는데 말 못 한 사람
34. 나와 정반대인 사람과 친해진 이야기
35. 낯선 사람에게 받은 작지만 큰 친절

**상상과 만약에** (가벼운 공상)
36. 한 달 동안 아무것도 안 해도 된다면
37. 하루만 다른 사람이 될 수 있다면
38. 내일 갑자기 100일 휴가가 생기면
39. 과거의 나에게 한마디 할 수 있다면
40. 로또 당첨되면 제일 먼저 할 일
41. 시간을 되돌릴 수 있다면 다시 가고 싶은 날
42. 전혀 다른 삶을 산다면 어떤 하루일까

**성장과 변화** (조금 깊은 이야기)
43. 나이 들면서 오히려 좋아진 것
44. 예전엔 몰랐는데 지금은 아는 것
45. 최근에 생각이 확 바뀐 것
46. 포기하고 나서 오히려 편해진 것
47. 요즘 나를 가장 많이 성장시키는 것
48. 올해 안에 꼭 해보고 싶은 것
49. 남들은 쉽게 하는데 나한테는 어려운 것
50. 지금의 나에게 해주고 싶은 말

#### 직접 작성
추천 주제 대신 자유롭게 텍스트를 입력할 수 있다.
- 글자 수 제한: 최대 30자 (예약 화면에서 한 줄로 보여야 하므로)

## 3. 시나리오별 동작

### 시나리오 1: 정상 플로우
1. 약수 5명, 망원 4명, 서촌 0명, 압구정 0명
2. 멤버 A가 서촌에 "대화 열기" 선택
3. 주제 입력: "요즘 빠져있는 넷플릭스 추천"
4. 서촌에 예약 완료 + 주제가 예약 화면에 노출
5. 멤버 B가 예약 화면에서 서촌의 주제를 보고 "나도 이거 이야기하고 싶다" → 서촌에 예약
6. 스팟 분산 성공

### 시나리오 2: 복수 빈 스팟에 각각 주제가 열림
1. 약수 5명, 망원 4명, 서촌 0명, 압구정 0명
2. 멤버 A가 서촌에 주제를 열고, 멤버 B가 압구정에 주제를 열음
3. 예약 화면에 서촌과 압구정에 각각 다른 주제가 보임
4. 다른 멤버들이 흥미 있는 주제 기준으로 스팟을 선택 → 분산 효과 극대화

### 시나리오 3: 첫 번째 예약자가 예약을 다른 스팟으로 변경
1. 멤버 A가 서촌에 주제를 열고 예약 완료
2. 멤버 B, C가 그 주제를 보고 서촌에 예약
3. 멤버 A가 예약을 약수로 변경하려 함
4. **안내 메시지 표시:**
   > "○○님이 열어주신 대화 주제로 다른 멤버들이 모였어요.
   > 💬 "요즘 빠져있는 넷플릭스 추천"
   > 이 대화를 기대하고 있는 멤버가 있는데, 그래도 옮기시겠어요?"
   >
   > [그래도 옮길게요] [이 스팟에 남을게요]
5. 옮기더라도 주제는 그대로 유지 (주제를 보고 모인 멤버들의 경험 보호)

### 시나리오 4: 주제가 열렸지만 아무도 추가 예약을 안 함
1. 멤버 A가 서촌에 주제를 열고 혼자 예약된 상태
2. 세션 날까지 추가 예약 없음
3. **특별한 처리 불필요** — 기존에도 1명만 예약된 스팟에서 세션이 진행되는 구조이므로 동일하게 처리

### 시나리오 5: 빈 스팟에 대화 열기 없이 그냥 예약하는 경우
1. 서촌이 0명이고 "대화 열기"가 활성화된 상태
2. 멤버가 서촌을 선택하지만 주제를 입력하지 않고 그냥 예약
3. **허용** — 주제 입력은 선택사항이지 필수가 아님
4. 이 경우 서촌은 "1명 예약 중"으로만 표시 (주제 없음)

### 시나리오 6: 대화 열기 활성화 후 인기 스팟이 4명 아래로 떨어짐
1. 약수 5명으로 인해 서촌에 대화 열기가 활성화됨
2. 약수에서 2명이 취소하여 약수 3명이 됨
3. **이미 열린 대화 주제는 유지** — 한번 열린 주제를 조건 변동으로 삭제하면 혼란
4. 다만 추가적인 빈 스팟에 새로 "대화 열기"가 활성화되지는 않음 (조건 미충족이므로)

### 시나리오 7: 모든 스팟에 예약이 있을 때
1. 약수 5명, 망원 4명, 서촌 2명, 압구정 1명
2. 0명인 스팟이 없으므로 "대화 열기" 기능은 어디에도 표시되지 않음

## 4. DB 변경사항

### 새 테이블: conversation_topics

```sql
CREATE TABLE IF NOT EXISTS conversation_topics (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  date TEXT NOT NULL,              -- 세션 날짜 (YYYY-MM-DD)
  spot TEXT NOT NULL,              -- 스팟 ID
  topic TEXT NOT NULL,             -- 대화 주제 (최대 30자)
  category TEXT,                   -- 주제 카테고리 (nullable, 직접 작성 시 null)
  created_by TEXT NOT NULL,        -- 주제를 연 멤버 이름
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(date, spot)              -- 한 세션(날짜+스팟)에 주제 1개만
);
```

### 기존 테이블 변경: 없음
- reservations 테이블 변경 불필요
- 대화 주제는 별도 테이블로 관리

## 5. API 엔드포인트

### POST /api/conversation-topics
대화 주제 생성

**Request:**
```json
{
  "date": "2026-04-09",
  "spot": "서촌_터틀도브",
  "topic": "요즘 빠져있는 넷플릭스 추천",
  "category": "일상 수다"
}
```

**검증 로직:**
1. 해당 날짜+스팟에 이미 주제가 있는지 확인 → 있으면 거부
2. 해당 날짜+스팟에 예약이 0명인지 확인 → 0명이 아니면 거부
3. 같은 날짜 다른 스팟 중 4명 이상인 곳이 있는지 확인 → 없으면 거부
4. topic 길이 30자 이내 확인

**Response (성공):**
```json
{
  "success": true,
  "topic": {
    "id": 1,
    "date": "2026-04-09",
    "spot": "서촌_터틀도브",
    "topic": "요즘 빠져있는 넷플릭스 추천"
  }
}
```

### GET /api/conversation-topics?date=2026-04-09
특정 날짜의 모든 대화 주제 조회

**Response:**
```json
{
  "topics": [
    {
      "date": "2026-04-09",
      "spot": "서촌_터틀도브",
      "topic": "요즘 빠져있는 넷플릭스 추천",
      "category": "일상 수다"
    },
    {
      "date": "2026-04-09",
      "spot": "압구정로데오_벤슨 테이스팅 라운지",
      "topic": "혼자 여행에서 생긴 에피소드",
      "category": "여행과 장소"
    }
  ],
  "eligible_spots": ["서촌_터틀도브", "압구정로데오_벤슨 테이스팅 라운지"]
}
```

`eligible_spots`: 현재 대화 열기 조건을 충족하는 스팟 목록 (0명이면서 다른 스팟에 4명 이상)

### GET /api/conversation-topics/suggestions
추천 주제 목록 반환 (카테고리별)

## 6. 프론트엔드 변경사항

### 6-1. SpotSelector 컴포넌트 수정

현재 SpotSelector에서 스팟별 예약 현황을 보여주고 있음. 여기에 다음을 추가:

1. **대화 열기 조건 충족 여부 체크**
   - GET /api/conversation-topics?date={date} 호출하여 eligible_spots 확인

2. **빈 스팟에 대화 열기 UI 표시**
   - eligible_spots에 포함된 스팟 중 주제가 아직 없는 스팟: "✨ 첫 번째로 예약하고, 나누고 싶은 대화를 열어보세요" 표시
   - 이미 주제가 있는 스팟: "💬 {주제}" 표시

3. **대화 열기 선택 시 주제 입력 모달/시트 표시**

### 6-2. 주제 입력 화면 (모달 또는 바텀시트)

```
어떤 이야기를 열어볼까요?

[카테고리 선택 탭]

[선택한 카테고리의 주제 목록]

○ 인생 최애 카페 or 단골집 자랑

...

[직접 작성하기]
입력 필드 (최대 30자)

[이 주제로 대화 열기] 버튼
```

### 6-3. 예약 변경 시 확인 모달

대화 주제를 연 멤버가 예약을 다른 스팟으로 변경하려 할 때:

```
○○님이 열어주신 대화 주제로 다른 멤버들이 모였어요.

💬 "요즘 빠져있는 넷플릭스 추천"

이 대화를 기대하고 있는 멤버가 있는데,
그래도 옮기시겠어요?

[그래도 옮길게요]  [이 스팟에 남을게요]
```

※ 해당 스팟에 본인 외 다른 예약자가 없는 경우에는 이 확인 없이 바로 변경 가능

---

## 7. 규칙 요약

| 항목 | 규칙 |
|------|------|
| 발동 조건 | 같은 날짜 다른 스팟 4명+ AND 해당 스팟 0명 |
| 주제 수 | 한 세션(날짜+스팟)당 1개 |
| 주제 작성 권한 | 해당 스팟 첫 번째 예약자만 |
| 주제 입력 | 카테고리에서 선택 또는 직접 작성 (최대 30자) |
| 주제 입력 강제 여부 | 선택사항 (주제 없이 예약도 가능) |
| 주제 작성자 이탈 시 | 주제 유지, 다른 예약자가 있으면 확인 메시지 |
| 조건 변동 시 | 이미 열린 주제는 유지, 새 대화 열기는 비활성화 |
| 이후 예약자 | 주제 추가/수정 불가, 열린 주제 보고 참여만 가능 |

---

## 8. 기대 효과

1. **스팟 분산**: 비인기 스팟에 대화 주제가 노출되면서 예약 유도
2. **멤버 경험 향상**: "사람 수"가 아니라 "대화 내용"으로 스팟을 선택하는 새로운 기준 제공
3. **자율성 유지**: 강제 없이 자발적 선택을 유도하는 넛지 구조
4. **익명 원칙 유지**: 주제만 노출, 작성자 정보 비공개
